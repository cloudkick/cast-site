Contributing to Cast

Cast is an open-source project released under the Apache 2.0 license so everyone is encouraged and welcome to contribute.

We accept many different kinds of contributions (code, tests, documentation, ideas, ..) which means that almost anyone is qualified to contribute.

If you need more information you are always welcome to join us on IRC or start a new thread on our mailing list where we are happy to help you!

Guidelines and rules for contributing

  • If you are about to implement a new feature or a bigger change, start a discussion on our mailing list first.

  • Open a new issue on our issue tracker for every change.

  • Create a new branch for every change, even if it is a one line bug fix (git makes branching cheap and easy). Branch name should be in the following format: <ticket_id>_branch_name

  • Every change must contain tests. Test coverage of 90% and above is preferred.

  • Stick to the Coding Guidelines.

  • After you are done with your change make sure that all the tests still pass.

  • After you think your modifications are ready, open a pull request on Github so others can review your code and provider further feedback.

Vagrant Development Image

To make participation and development easier we have created a Vagrant image which contains everything required for you to start developing.

For more information about getting and using the Vagrant image, please see Using Vagrant for development page (the whole process of downloading and setting up the image is very easy and straight-forward and it should not take more than 10 minutes).
